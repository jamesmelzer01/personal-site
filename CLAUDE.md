## Project Overview

Personal site for James Melzer (jamesmelzer.com). Testbed for a Figma-to-code design token pipeline using the Figma Plugin API, Style Dictionary, and GitHub Actions.

**Hosting:** Vercel — auto-deploys on push to `main`.
**Repo:** https://github.com/james-bearings-ux/personal-site
**Branch strategy:** Feature branches → PR → merge to `main` at working milestones only.

## Dev Commands

```bash
npm run dev          # local dev server (localhost:3000)
npm run build        # production build
npm run lint         # ESLint
npm run build-tokens # Style Dictionary: tokens/figma.raw.json → src/styles/tokens.css
npm run sync-tokens  # fetch-tokens + build-tokens in sequence
```

Node managed via nvm. Run `nvm use` if versions mismatch.

## Architecture

**Framework:** Next.js 16, App Router, TypeScript.
**CSS:** Hand-rolled CSS custom properties — no Tailwind. All design values sourced from `src/styles/tokens.css`.

### Token Pipeline

```
Figma Variables (source of truth)
  → Figma plugin          (reads variables via Plugin API, copies JSON to clipboard)
  → tokens/figma.raw.json (paste from clipboard, commit — this triggers the rest)
  → style-dictionary.config.mjs  (transforms DTCG JSON → CSS custom properties)
  → src/styles/tokens.css        (generated, do not edit by hand)
  → src/app/layout.tsx           (imports tokens.css globally)
```

`scripts/fetch-tokens.mjs` is a REST API alternative to the plugin — written but blocked on Figma Enterprise plan (`file_variables:read` scope required).

### Key files

- `src/app/layout.tsx` — root layout, imports `globals.css` and `tokens.css`
- `src/app/globals.css` — CSS reset only
- `src/styles/tokens.css` — generated by Style Dictionary, do not edit by hand
- `tokens/figma.raw.json` — DTCG-format token source, updated via Figma plugin
- `scripts/fetch-tokens.mjs` — Figma REST API fetch (Enterprise only, not in active use)
- `style-dictionary.config.mjs` — Style Dictionary config
